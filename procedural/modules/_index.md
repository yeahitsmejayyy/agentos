# modules — Promoted code + its tests

The high-fidelity rung: real, finished code with **its own tests**. There's nothing left to interpret
— so a module is the *standard* and its *oracle* in one package. **`memory_type: procedural`**,
`kind: module`, **`materializable: true`** (install the code into a project; the installer is Part B).

Each module is a directory: `modules/<name>/` with a `README.md` (the standard, often the prose note
it graduated from), `src/` (the code), and `tests/` (the bundled oracle — a module isn't proven
without it). Copy `_template/` to start one. Ships empty — modules are *promoted* from real work via
the gated `promote-standard` gesture, never authored cold.
